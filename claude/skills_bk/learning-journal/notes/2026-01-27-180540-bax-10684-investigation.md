## 発見した問題

### 問題1: グループ削除時にグループ経由メンバーが無効化されない（BAX-10684ブランチで修正中・未マージ）
- **ファイル**: `backend/task/user_deleted.go`
- **問題**: ユーザーグループが削除された時、`DisableMemberByUserUniqueCode`が`chatmember_related_to IS NULL`条件を持っていたため、グループ経由メンバーが無効化されなかった
- **修正コミット**: ccc2c84（ブランチ: `origin/BAX-10684`）
- **ステータス**: **未マージ**（developにマージされていない）

### 新たに発見した問題
- **ファイル**: `backend/infrastructure/postgresql/background_task.go:434-451`
- **関数**: `GetRoomsByMemberTargetUniqueCode`
- **問題**: `chatmember_is_disabled = 0`の条件がないため、削除済みのルームも取得してしまう

```go
// 現在のコード（バグあり）
tx := db.Model(&ChatMember{}).
    Preload("ChatRoom").
    Preload("ChatMemberRole").
    Where("chatmember_target_unique_code = ?", uuc).
    Find(&members)
// chatmember_is_disabled = 0 の条件がない！
```

### 発生メカニズム

1. グループがルームから削除される（chatmember_is_disabled = 1）
2. その後、新しいメンバーがグループに追加される
3. `USER_GROUP_MEMBER_INSERTED`タスクが発動
4. `GetRoomsByMemberTargetUniqueCode`で削除済みルームも取得
5. 削除済みルームに対してもメンバーが追加されてしまう

## 修正案

```go
// 修正後
tx := db.Model(&ChatMember{}).
    Preload("ChatRoom").
    Preload("ChatMemberRole").
    Where("chatmember_target_unique_code = ?", uuc).
    Where("chatmember_is_disabled = 0").  // これを追加
    Find(&members)
```

## 結論

BAX-10684は2つの異なるバグが複合していた：

1. **未マージ**: グループ削除時にグループ経由メンバーが無効化されない（`user_deleted.go`）
2. **未修正**: グループ追加時に削除済みルームにもメンバーが追加される（`GetRoomsByMemberTargetUniqueCode`）

今回のケースは2番目のバグに該当。両方の修正とマージが必要。
