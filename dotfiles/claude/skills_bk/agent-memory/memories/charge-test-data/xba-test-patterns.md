## パターン詳細

### D-01+B-01: 初回年払い
- 無料期間終了後、初回の年払い課金が発生する状態
- 既存の課金データなし

### B-02: 年払い継続更新
- 年払い契約が満了し、年払いで継続更新される状態

### B-03: 年払い→月払い変更
- 年払い契約が満了し、月払いに変更される状態

### D-02: 契約満了日またぎ
- 契約満了日の23:59→0:01をまたぐ状態
- 日付境界での契約更新処理確認

### D-03: 年払い残り1ヶ月
- 年払い契約11ヶ月目（残り1ヶ月）の状態
- 契約満了間近での退会予約動作確認

### D-04+B-04: 契約満了月1日
- 年払い契約満了月の1日の状態
- 契約更新確認メール・イベント生成確認

### D-05+F-01: 月払い→年払い変更
- 月払い→年払い変更予約後、契約満了の状態
- 2企業：年払いONのまま / 年払いOFF（強制月払い）

### D-06+F-01: 年払い→月払い変更
- 年払い→月払い変更予約後、契約満了の状態
- 2企業：年払いONのまま / 年払いOFF

### X-04: 年払い2年目更新直前
- 年払い2年目更新直前の状態
- 2回目以降の更新処理確認
- 課金データを2つ作成（1年目と2年目）

### K-02: 課金開始日未来日
- 年払い満了→月払い変更＋課金開始日が未来日の状態
- 契約終了後の無料期間（再無料期間）発生時の挙動確認

### CH-01: 課金無効
- 年払い契約満了月＋課金無効の状態
- 課金無効時に契約更新確認メールが送信されないことを確認
