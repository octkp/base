# Work Report Generator

`docs/takano` 配下のgitコミット履歴から、指定期間の作業レポートを生成するスキル。

## ベースパス

`/Users/takano_y/ghq/github.com/kokopelli-inc/badev-knowledge-base`

docs/takanoのパス: `{ベースパス}/docs/takano`

## ワークフロー

### 1. 期間指定の解釈

ユーザーの自然言語入力をgit logオプションに変換する。

| 入力例 | --since | --until |
|--------|---------|---------|
| 昨日 | `yesterday` | `today` |
| 今日 | `today` | `now` |
| 今週 | `last monday` | `now` |
| 先週 | `2 weeks ago monday` | `last monday` |
| 今月 | 当月1日（例: `2026-02-01`） | `now` |
| 先月 | 前月1日（例: `2026-01-01`） | 当月1日（例: `2026-02-01`） |
| 日付範囲（YYYY-MM-DD〜YYYY-MM-DD） | 開始日 | 終了日 |

**注意**: 「先月」「今月」は現在の日付から月初・月末を計算すること。`date` コマンドで現在日時を取得する。

### 2. git logで変更ファイル一覧を取得

```bash
cd {ベースパス} && git log --format="%H %ad" --date=short --name-only --since="..." --until="..." -- docs/takano
```

### 3. ファイルパスからチケット・カテゴリを抽出

パス構造: `docs/takano/{カテゴリ}/{チケット番号}_{説明}/`

カテゴリ一覧:
- `調査` - バグ調査、原因特定、影響調査
- `実装` - 機能実装、修正、リファクタリング
- `勉強会` - 勉強会資料、学習ノート
- `その他` - 上記に当てはまらないもの

抽出ルール:
- パスの第3セグメント（`docs/takano/` の次）がカテゴリ
- 第4セグメントからチケット番号と説明を分離（`BAX-XXXX_説明` → チケット番号: `BAX-XXXX`, 説明: `説明`）
- チケット番号がないもの（勉強会資料など）はファイル名をそのまま使う
- 同じチケットが複数カテゴリに存在する場合（調査→実装など）、それぞれ別項目として扱う
- `.DS_Store` やその他の隠しファイルは除外する

### 4. 各チケットのREADME.mdから概要を取得

各チケットディレクトリの `README.md` を読み取り、以下のセクションを抽出:

1. **概要** (`## 概要` セクション)
2. **結論/解決策** (`## 結論/解決策` セクション)

README.mdが存在しない場合やセクションが見つからない場合は、ディレクトリ名の説明部分をそのまま使用する。

勉強会カテゴリの場合はファイル名（`.md`を除く）をタイトルとして使用する。

### 5. レポート生成

以下のフォーマットでレポートを生成する。

```markdown
# 作業レポート: {期間の表示文字列}

生成日時: {YYYY-MM-DD HH:MM}

## サマリー

- 調査: X件
- 実装: X件
- 勉強会: X件
- その他: X件

## 調査

### BAX-XXXX: {チケット名/説明}
{README.mdの概要から抽出した内容}

**結論/解決策**: {README.mdの結論セクションから抽出。なければ省略}

## 実装

### BAX-XXXX: {チケット名/説明}
{README.mdの概要から抽出した内容}

## 勉強会

### {タイトル}
{ファイル名から抽出}

## その他

### {タイトル}
{内容}
```

**注意**:
- 0件のカテゴリはサマリーに含めるが、セクション自体は省略する
- 生成日時は `date "+%Y-%m-%d %H:%M"` で取得する

## 出力先

### 画面表示

レポート内容をそのまま会話に出力する。

### ファイル保存

`docs/takano/レポート/` ディレクトリに保存する（存在しない場合は作成）。

ファイル名の規則:

| 期間 | ファイル名の例 |
|------|---------------|
| 昨日（特定日） | `2026-02-18.md` |
| 今日（特定日） | `2026-02-19.md` |
| 今週 | `2026-02-16_2026-02-22.md`（月曜〜日曜の日付範囲） |
| 先週 | `2026-02-09_2026-02-15.md`（月曜〜日曜の日付範囲） |
| 今月 | `2026-02.md` |
| 先月 | `2026-01.md` |
| 日付範囲 | `2026-02-01_2026-02-15.md` |

週のファイル名は `date` コマンドで当該週の月曜と日曜の日付を算出して使用する。

## 使用例

### 今週のレポート

ユーザー: 「今週のレポート」

実行:
1. `date` で現在日時取得
2. `git log --since="last monday" --until="now" -- docs/takano` で変更ファイル一覧取得
3. ファイルパスからチケット・カテゴリ抽出
4. 各README.md読み取り
5. レポート生成・表示
6. `docs/takano/レポート/2026-02-16_2026-02-22.md` に保存

### 先月のレポート

ユーザー: 「先月のレポート」

実行:
1. 現在日付から先月の範囲を計算（例: 2026-01-01 〜 2026-02-01）
2. `git log --since="2026-01-01" --until="2026-02-01" -- docs/takano`
3. 以下同様

### 日付範囲指定

ユーザー: 「2026-02-01〜2026-02-15のレポート」

実行:
1. `git log --since="2026-02-01" --until="2026-02-15" -- docs/takano`
2. 以下同様

## 注意事項

- コミットメッセージは簡素（"c", "commit"等）なので**参照しない**。ファイルパスのみを使う
- 同じチケットで複数コミットがある場合は1つにまとめる（重複排除）
- `.DS_Store` や `.claude/` 配下のファイルは除外する
- README.mdの読み取りに失敗した場合はディレクトリ名の説明で代替する
