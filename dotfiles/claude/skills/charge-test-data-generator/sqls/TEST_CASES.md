# 年払い課金テストケース一覧

## 1. 月払い課金テスト

| テストID | テスト目的 | 期待結果 |
|----------|-----------|----------|
| TC-001 | 通常の月払い課金作成 | 課金データが1件作成される |
| TC-002 | 無料期間中の企業 | 課金データが作成されない |
| TC-003 | 退会予定企業 | 課金データが作成されない |

## 2. 年払い課金テスト

| テストID | テスト目的 | 期待結果 |
|----------|-----------|----------|
| TC-101 | 初回年払い課金作成 | 年払い課金が作成され、company_charge_typeがANNUALに更新 |
| TC-102 | 年払い2ヶ月目のオプション課金のみ作成 | オプション課金のみ作成、プラン料金は含まれない |
| TC-103 | 年払い契約期間終了後の更新（年払い継続） | 新しい年払い課金が作成される |
| TC-104 | プラン変更（年払い → 月払い） | 月払い課金が作成され、company_charge_typeがMONTHLYに更新 |
| TC-105 | 月払い → 年払いへのアップグレード | 年払い課金が作成される |
| TC-106 | 年払い期間中のプランアップグレード | 次回更新時に新プランが反映 |
| TC-107 | 年払い期間中のプランダウングレード | 次回更新時に新プランが反映 |

## 3. オプション課金テスト

## テストケース優先度

### 高優先度（必須）
- TC-001, TC-002, TC-003（月払い基本）
- TC-101, TC-102, TC-103, TC-104（年払い基本）
- TC-115（空課金防止）
- TC-201, TC-203（課金実行・2重課金防止）
- TC-301, TC-302（境界値基本）

### 中優先度（推奨）
- TC-105（月払い→年払いアップグレード）
- TC-108, TC-109, TC-110（オプション課金）
- TC-113（年払い期間中退会）
- TC-303（税込価格計算）
- TC-401（メール送信）

### 低優先度（あれば良い）
- TC-106, TC-107（プラン変更次回反映）
- TC-111, TC-112（オプション詳細）
- TC-114（退会キャンセル）
- TC-202, TC-204, TC-205, TC-206（異常系詳細）
- TC-304, TC-305, TC-306, TC-307, TC-308（境界値詳細）
