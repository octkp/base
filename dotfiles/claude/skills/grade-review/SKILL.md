# Grade Review Skill

等級評価の自己評価文をレビューするスキル。

## 使用方法

```
/grade-review [等級] [評価軸]
```

### パラメータ

| パラメータ | 説明 | 例 |
|-----------|------|-----|
| 等級 | G1〜G9 | G3, G4, G5 |
| 評価軸 | 評価する軸（省略時は全軸） | 目標達成, 課題解決, 組織貢献, 専門性 |

### 例

```
/grade-review G3                    # G3の全評価軸をレビュー
/grade-review G4 目標達成           # G4の目標達成のみレビュー
/grade-review G5 課題解決 組織貢献  # G5の複数軸をレビュー
```

## 評価軸と対応ファイル

| 評価軸 | ファイル |
|--------|----------|
| 目標達成 | `~/base/docs/grade/目標達成.md` |
| 課題解決 | `~/base/docs/grade/課題解決.md` |
| 組織貢献 | `~/base/docs/grade/組織貢献.md` |
| 専門性 | `~/base/docs/grade/専門性.md` |

## ガイドライン参照

ガイドラインファイル: `~/base/docs/grade/.GUIDELINE.md`

## 会社MVV（Mission, Vision, Value）

自己評価文（特に「組織貢献」）では、以下のMVVとの整合性も確認する。

| 項目 | 内容 |
|------|------|
| **ミッション** | 企業価値の中に、未来を見つける |
| **ビジョン** | 中小企業にテクノロジーを届ける |

### MVV実践の観点

等級ごとにMVVへの関わり方が異なる：

| 等級 | MVVとの関わり |
|------|---------------|
| G1 | MVVを把握し挑戦する |
| G2 | MVVを理解し実践する |
| G3 | MVVを理解し積極的に実践する |
| G4 | MVVを理解・実践し、Grメンバーに伝達する |
| G5 | Grメンバーの強みを活かしながらMVVを実践 |
| G6 | 部門全体がMVVを実践できるよう適切なアドバイス・支援 |
| G7 | MVVを実践しながら、企業成長を促す組織風土創りを担う |
| G8 | MVVを実践しながら、企業成長を促す組織風土創りを率先して担う |
| G9 | MVVを実践しながら、経営状況に最適な組織風土を常に検討し磨きあげる |

### MVV実践の具体例

#### ミッション「企業価値の中に、未来を見つける」との整合
- 顧客企業の潜在的な価値を発見し、提案した
- データ分析を通じて顧客の成長機会を特定した
- 新規サービス/機能で顧客の事業発展に貢献した

#### ビジョン「中小企業にテクノロジーを届ける」との整合
- 使いやすいUI/UXを実現し、テクノロジー導入のハードルを下げた
- 中小企業の課題をテクノロジーで解決するソリューションを開発した
- 顧客の業務効率化・DX推進を支援した

## レビュー観点

### 1. 等級基準との整合性

指定された等級のガイドライン定義と自己評価文を比較し、以下を確認する：

- **責任範囲**: 等級に応じた責任範囲（担当業務/Gr/部門/管掌領域/全社）で記述されているか
- **期待役割**: その等級で期待される役割・行動が反映されているか
- **キーワード**: ガイドライン内の重要キーワードが適切に使用されているか

### 2. 定量的表現のチェック

具体的な数値や指標で成果を示しているか確認する：

#### 良い例
- 「月間売上を前年比120%に向上させた」
- 「バグ件数を50件から10件に削減した」
- 「顧客満足度スコアを3.5から4.2に改善した」
- 「3つの新機能をリリースした」
- 「チームメンバー5名を育成した」

#### 改善が必要な例
- 「売上を向上させた」（数値なし）
- 「バグを削減した」（程度が不明）
- 「顧客満足度を改善した」（具体性なし）

### 3. 定性的表現のチェック

質的な成果や取り組みの深さを示しているか確認する：

#### 良い例
- 「部門横断プロジェクトでリーダーシップを発揮し、関係者間の合意形成を主導した」
- 「新規メンバーのオンボーディングプロセスを再設計し、早期戦力化に貢献した」
- 「技術的負債の解消方針を策定し、チーム全体の開発効率向上に寄与した」

#### 改善が必要な例
- 「頑張った」（具体性なし）
- 「良い結果を出した」（内容が不明）
- 「コミュニケーションを取った」（どのように、何のためかが不明）

## 等級別チェックポイント

### G1（エントリーメンバー）
- 自己の目標達成について記述しているか
- 担当業務の課題発見・対応について記述しているか
- 関連メンバーとの関係構築について記述しているか

### G2（ジュニアメンバー）
- 自己の目標達成を通じたGr貢献について記述しているか
- 課題の類型化・既存解決策の応用について記述しているか
- 相互協力について記述しているか

### G3（ミドルメンバー）
- 自己＋メンバーの目標達成支援について記述しているか
- 関係者との協力による課題解決について記述しているか
- 効果的なコミュニケーション・相互協力について記述しているか

### G4（シニアメンバー）
- 担当Grの計画達成への責任について記述しているか
- 課題の情報展開・再発防止について記述しているか
- メンバーへのフィードバック・成果支援について記述しているか

### G5（グループ責任者）
- 担当Grの計画達成責任＋他チーム支援について記述しているか
- 解決策の標準化について記述しているか
- Grメンバーの強み活用・育成について記述しているか

### G6（グループ責任者・上位）
- 担当Grの計画立案・達成責任について記述しているか
- 将来課題の察知・未然防止について記述しているか
- 事業部門での主体的コミュニケーションについて記述しているか

### G7以上
- 部門/管掌領域/全社レベルの責任について記述しているか
- 中長期課題への取り組みについて記述しているか
- 組織づくり・人材育成への貢献について記述しているか

## 注意事項

- ガイドラインの「マネジメント」はマネジメントコース向け、「高度専門性」はエキスパートコース向けの評価軸
- 本スキルでの「専門性」は主に「高度専門性（エキスパートコース）」を指す
- 等級が上がるほど、より広い責任範囲と中長期的な視点が求められる
