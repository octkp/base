# JIRA Slack Activity Summary

SlackチャンネルのJIRA通知メッセージを解析し、誰が何をやったかをまとめるスキル。

## 対象チャンネル

- チャンネルID: `C0AHLMY1W04` (#ba-jira)
- このチャンネルにはJIRAの操作通知のみが投稿される

## ワークフロー

### 1. ユーザー入力の解釈

ユーザーから以下の情報を受け取る:

- **期間** (必須): 自然言語で指定（例: 今日、昨日、今週、先週、今月、2026-03-01〜2026-03-04）
- **対象者** (任意): 特定の人のみをフィルタ（例: 三島さん、井上さん、全員）

### 2. 期間をUnixタイムスタンプに変換

Slack APIの `oldest` / `latest` パラメータはUnixタイムスタンプ（秒）で指定する。
`date` コマンドで変換する。

| 入力例 | oldest | latest |
|--------|--------|--------|
| 今日 | 当日 00:00:00 JST | 現在時刻 |
| 昨日 | 前日 00:00:00 JST | 当日 00:00:00 JST |
| 今週 | 今週月曜 00:00:00 JST | 現在時刻 |
| 先週 | 先週月曜 00:00:00 JST | 今週月曜 00:00:00 JST |
| 今月 | 当月1日 00:00:00 JST | 現在時刻 |
| 先月 | 前月1日 00:00:00 JST | 当月1日 00:00:00 JST |
| 日付範囲 | 開始日 00:00:00 JST | 終了日+1日 00:00:00 JST |

**macOS (BSD date) でのタイムスタンプ取得例:**

```bash
# 今日の00:00 JST
date -j -f "%Y-%m-%d %H:%M:%S" "$(date +%Y-%m-%d) 00:00:00" "+%s"

# 昨日の00:00 JST
date -j -v-1d -f "%Y-%m-%d %H:%M:%S" "$(date -v-1d +%Y-%m-%d) 00:00:00" "+%s"

# 現在時刻
date "+%s"

# 特定の日付
date -j -f "%Y-%m-%d %H:%M:%S" "2026-03-01 00:00:00" "+%s"
```

### 3. Slackからメッセージを取得

`mcp__plugin_slack_slack__slack_read_channel` を使用してメッセージを取得する。

パラメータ:
- `channel_id`: `C0AHLMY1W04`
- `oldest`: 開始タイムスタンプ（文字列）
- `latest`: 終了タイムスタンプ（文字列）
- `limit`: `100`
- `response_format`: `detailed`

**重要**: 100件を超える場合は `cursor` を使ってページネーションで全件取得する。pagination_infoに`next_cursor`がある限り繰り返す。

### 4. メッセージのパース

全メッセージはJira bot（`U01J5B51WJH`）からの投稿。以下のパターンで解析する。

#### 4.1 人物名の抽出

2つの形式がある:

1. **Slackメンション形式**: `<slack://user?team=...&id=UXXXXXXXX|@表示名>` → 「表示名」を抽出
2. **プレーンテキスト形式**: `*名前 updated...` → 先頭の人名を抽出

**「Automation for Jira」は人物ではないので除外する。**

#### 4.2 アクション種別の抽出

メッセージ1行目のパターンから種別を判定:

| パターン | アクション |
|---------|-----------|
| `{person} created a {type}` | チケット作成 |
| `{person} transitioned a {type} from [A] ⟶ [B]` | ステータス変更 |
| `{person} updated the {field} of a {type}` | フィールド更新 |
| `{person} commented on a {type}` | コメント |
| `{person} assigned a {type} from A ⟶ B` | 担当者変更 |

#### 4.3 チケット情報の抽出

メッセージ2行目からチケット番号とタイトルを抽出:

`*<https://kokopelli-inc.atlassian.net/browse/BAX-XXXX?...|BAX-XXXX タイトル>*`

→ チケット番号: `BAX-XXXX`、タイトル: 残りのテキスト

#### 4.4 ステータス遷移の抽出

`transitioned` メッセージの場合、遷移元・遷移先のステータスを抽出:

`` `[ステータスA]` ⟶ `[ステータスB]` ``

### 5. 対象者フィルタリング

対象者が指定されている場合、パースした人物名に対して部分一致でフィルタする。

- 「三島さん」→ 「三島」で部分一致検索
- 「井上さん」→ 「井上」で部分一致検索
- 「Yukioka」→ 「Yukioka」で部分一致検索

「さん」「くん」等の敬称は除去してから検索する。

### 6. データの集約

パースしたデータを以下のルールで集約する:

1. **人物ごとにグループ化**
2. **同一チケットの操作をまとめる**: 同じチケットに対する複数の `updated` はまとめて「N回更新」とする
3. **意味のある操作を優先表示**:
   - チケット作成 (created)
   - ステータス変更 (transitioned) ← 最も重要
   - コメント (commented)
   - 担当者変更 (assigned)
   - フィールド更新 (updated) ← 回数だけ表示

### 7. レポート生成

以下のフォーマットで出力する:

```markdown
# JIRA活動サマリー: {期間の表示文字列}

生成日時: {YYYY-MM-DD HH:MM}
対象チャンネル: #ba-jira

{対象者フィルタがある場合: "対象者: {名前}"}

## {人物名1}

### {BAX-XXXX}: {チケットタイトル}
- ステータス変更: [A] → [B]
- コメント: 1件
- フィールド更新: 3回（詳細, Sprint, Attachment）

### {BAX-YYYY}: {チケットタイトル}
- チケット作成
- ステータス変更: [TODO] → [進行中]

---

## {人物名2}

### {BAX-ZZZZ}: {チケットタイトル}
- ステータス変更: [レビュー中] → [クローズ]

---

## 統計

| 名前 | 作成 | ステータス変更 | コメント | 更新 | 担当者変更 | 合計 |
|------|------|---------------|---------|------|-----------|------|
| {名前1} | X | X | X | X | X | XX |
| {名前2} | X | X | X | X | X | XX |
| **合計** | **X** | **X** | **X** | **X** | **X** | **XX** |
```

**注意**:
- 活動がない人は表示しない
- 人物名は表示名そのまま使用（Slackメンションの`@`は除去）
- チケットタイトルからリンクURL (`https://kokopelli-inc.atlassian.net/browse/BAX-XXXX`) を付与する
- 統計テーブルは活動数の多い人順にソートする

## 出力

レポートを会話に出力するのみ。ファイル保存はしない。

## 使用例

### 今日の全員の活動

ユーザー: 「今日のJIRA活動まとめて」

### 特定の人の今週の活動

ユーザー: 「三島さんの今週の活動を教えて」

### 先週のまとめ

ユーザー: 「先週みんな何やってた？」

### 日付範囲指定

ユーザー: 「2026-03-01から2026-03-04の井上さんの活動」

## 注意事項

- Slack APIのlimitは最大100件/リクエスト。期間が長い場合はページネーションが必要
- 「Automation for Jira」の操作は人の活動ではないため集計対象外とする
- 同一人物でも表記揺れがある可能性がある（漢字名 vs ローマ字名、表示名変更等）。パース時に気づいたら統合を試みる
- 大量メッセージ（数百件以上）の場合は取得に時間がかかるため、進捗を随時報告する
