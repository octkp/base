# 組織貢献

## 1. 複数リポジトリのオーナーとしてメンバーのサポート

**課題**

- 複数リポジトリにまたがる開発において、レビュワーが不足しレビュー待ちが発生することで開発が停滞するリスクがあった

**取り組み**

- 担当リポジトリのPRは即日レビューを徹底し、メンバーがブロックされない体制を維持した
- メンバーが開発に集中できるようブランチワークやマージフローを整備した
- 複数リポジトリ間の依存やコンフリクトを自ら調整し、メンバーの手戻りを防いだ
- レビューで見るべき観点をメンバーに共有し、セルフチェックの質を向上させた

**成果**

- 5つのリポジトリのオーナーとして、メンバーがレビュー待ちでブロックされることなくスムーズに開発を進められる体制を維持した

## 2. CIによる静的解析の導入

**課題**

- 10リポジトリ中、5リポジトリにCIでの静的解析が入っておらず、品質が担保されていない状態だった

**取り組み**

- ba-site-backend、ba-site-frontend、ba-fukuri、ba-admin-frontendにCI/CDパイプラインを構築
- 静的解析を組み込み、一定の基準を満たさないコードがリリースされない仕組みを整備
- メンバーが自分のコードの品質を早期にフィードバックとして受け取れる環境を構築した

**成果**

- 4つのリポジトリにCIによる静的解析を導入し、コード品質の担保を自動化した
- メンバーがPR作成時に即座にフィードバックを得られるようになり、レビュー前の品質が向上した

## 3. postgres障害対応

**課題**

- postgresアップデート翌日に障害が発生し、サービスに影響があるかわからない状態だった

**取り組み**

- 主体的にSRE、PdMと密に連携し調査を進めた
- 影響のあるクエリを特定し、リファクタリングを実施した
- 障害の原因・影響範囲・対応内容をメンバーに共有し、同様の事象への備えとして知見を展開した

**成果**

- 3時間ほどで暫定対応を即座に実施した
- 影響範囲を特定し、問題ないことを関連メンバーに共有した
- 負荷の高いクエリのリファクタリングを実施し、リリースした
